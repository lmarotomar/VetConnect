## Logros de esta sesión

### 1. Flujo completo funcional (end-to-end)
Registro → Login → Dashboard → Crear cliente+mascota → Crear cita → Ver citas → Logout

Todos los datos se guardan y leen desde Supabase real. Sin localStorage.

---

### 2. Bugs corregidos

| Bug | Archivo | Fix |
|-----|---------|-----|
| `supabase.from is not a function` (root cause) | `auth/supabase-client.js` | `let supabase` → `let _sb`; CDN conflicto de variables |
| Logout loop infinito | `index.html`, `auth/login.html` | `supabaseClient.auth.signOut()` + `localStorage.removeItem(tokenKey)` |
| Usuario sin nombre/rol en header | `index.html` | `ROLE_LABELS` definido inline en `updateUserDisplay()` |
| Modal no cerraba tras crear cliente | `modules/dashboard.js` | `App.closeModal()` antes de `App.refreshData()` |
| Appointments: FECHA vacía, HORA "undefined", CLIENTE/MASCOTA "N/A" | `modules/appointments.js`, `modules/dashboard.js` | Field mapping: `appointment_date`, `appointment_time`, objetos anidados `client`/`patient` |
| RLS bloqueaba todos los inserts | Supabase SQL Editor | Políticas permisivas para `authenticated` en todas las tablas |
| Módulos con números falsos hardcodeados | `inventory.js`, `reports.js`, `communication.js`, `billing.js` | Reemplazados por datos reales o ceros honestos |
